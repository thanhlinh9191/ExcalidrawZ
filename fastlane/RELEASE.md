# Fastlane Release Workflow

This folder manages App Store Connect metadata, App Store screenshots, and Sparkle release notes. It does not upload builds unless a lane is explicitly changed to do so.

For exact commands, use `COMMANDS.md`. `README.md` is generated by fastlane and is only a lane list.

## Release Flow

1. Archive and upload the build from Xcode Organizer.
2. Update platform release notes:

```text
metadata-ios/<locale>/release_notes.txt
metadata-mac/<locale>/release_notes.txt
```

3. Regenerate screenshots if the App Store preview images changed.
4. Run the relevant dry-run command from `COMMANDS.md`.
5. Upload metadata or iOS release assets with the matching version.

The lanes read the current Xcode marketing version when `version:` is omitted, but passing the version explicitly is clearer during release preparation.

## Metadata

Metadata is staged from a shared baseline plus platform overrides:

```text
metadata/       # shared name, subtitle, keywords, promo text, description
metadata-ios/   # iOS/iPadOS overrides, especially release_notes.txt
metadata-mac/   # macOS overrides, especially release_notes.txt
```

Keep `release_notes.txt` platform-specific. Do not mention iOS-only changes in macOS release notes, or macOS-only changes in iOS release notes.

Shared metadata files per locale:

```text
name.txt
subtitle.txt
keywords.txt
promotional_text.txt
description.txt
```

Platform-specific metadata files per locale:

```text
release_notes.txt
```

Current shared metadata locales:

```text

pl, pt-BR, ru, th, tr, vi, zh-Hans, zh-Hant
```

## Authentication

Authentication uses an App Store Connect API key with App Manager access.

Create a local ignored file at `fastlane/.env.local`:

```env
APP_STORE_CONNECT_API_KEY_ID=...
APP_STORE_CONNECT_API_ISSUER_ID=...
APP_STORE_CONNECT_API_KEY_PATH=./fastlane/AuthKey_XXXXXXXXXX.p8
```

Keep `.env.local` and `.p8` key files out of git.

## Sparkle

Sparkle release notes are generated from the staged macOS metadata source, so `metadata-mac/<locale>/release_notes.txt` is the source of truth.

The Sparkle lane writes localized HTML files under:

```text
WebPage/public/downloads/
```

If the local appcast already contains an item for the version, the lane patches localized `sparkle:releaseNotesLink` entries. Run the normal appcast generation flow before this lane when publishing a new non-App Store build.

## Screenshots

Preview screenshots are generated from a text-free source image plus localized text config.

Source images:

```text
previews/assets/iphone/AppStore_iPhone_Previews.png
previews/assets/ipad/AppStore_iPad_Previews.png
```

Optional localized source images use the same basename with a locale suffix, such as:

```text
previews/assets/ipad/AppStore_iPad_Previews-zh-Hans.png
```

Text configs:

```text
previews/iphone.json
previews/ipad.json
```

The renderer writes preview strips to `previews/output/`, then the splitter writes App Store screenshots to `screenshots/<locale>/`.

Use `sliceCount` in the preview config when a source strip contains more design area than the screenshots you want to upload. Use `gapWidth` only when the source strip intentionally includes visual spacing between screenshots; the splitter skips the gap so final screenshots do not contain baked-in spacing.

App Store screenshots should be opaque PNGs. If App Store Connect reports alpha/transparency, check the source image and generated screenshot with:

```sh
sips -g hasAlpha path/to/image.png
```

Generated screenshot outputs are ignored by git.
