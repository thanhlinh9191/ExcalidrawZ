# Fastlane Agent Notes

- `README.md` is generated by fastlane. Put durable workflow notes in `RELEASE.md` or this file.
- `metadata/` is shared App Store metadata. Keep platform release notes in `metadata-ios/<locale>/release_notes.txt` and `metadata-mac/<locale>/release_notes.txt`.
- Do not mention iOS-only changes in macOS release notes, or macOS-only changes in iOS release notes.
- Use explicit platform lanes by default: `fastlane upload_metadata platform:mac version:x.y.z`, `fastlane upload_metadata platform:ios version:x.y.z`, or `fastlane upload_ios_release_assets version:x.y.z`.
- Sparkle release notes are generated from the staged macOS metadata source.
- Generated outputs and secrets should stay out of git: `.generated/`, `previews/output/`, `screenshots/`, `.env.local`, and App Store Connect `.p8` keys.
